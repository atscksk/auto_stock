# 무한매수 V4 개선 개발 체크리스트

이 문서는 현재 무한매수 V4 구현을 실거래 가능한 수준으로 끌어올리기 위한 단계별 개발 계획이다.

체크박스는 작업이 끝날 때마다 직접 갱신한다.

## 0단계: 현재 상태 고정

- [x] 현재 구현 범위를 명확히 문서화한다.
- [x] 현재 미구현 기능을 README 또는 전략 문서에 표시한다.
- [x] 실거래 금지 상태를 문서에 명시한다.
- [x] 현재 백테스트 결과를 기준 스냅샷으로 저장한다.
- [x] TQQQ, SOXL 각각의 2025 H1, 2025 H2, 2026 H1 백테스트 결과를 기록한다.

완료 기준:

```text
현재 구현이 MVP인지, 실거래 가능 버전인지 혼동되지 않아야 한다.
```

## 1단계: 전략 엔진 안전성 보강

- [x] NORMAL_BACK에서 별표%가 음수일 때 평단 이하 매도를 막는다.
- [x] 별표 매도가가 평균단가보다 낮으면 매도거부로 전환한다.
- [x] 매도거부 상태와 사유를 계획 결과에 명확히 남긴다.
- [x] 매도거부가 연속으로 발생할 때의 제한 규칙을 구현한다.
- [x] 매도거부 상태에서도 15% 보조 매도 유지 여부를 명확히 처리한다.
- [x] 매수거부 사유를 코드화한다.
- [x] 매수거부 시 신규 매수 차단과 기존 매도 유지 동작을 분리한다.
- [x] 현금 부족, 평균단가 개선 효과 부족, 추세 필터 차단을 각각 다른 사유로 기록한다.

완료 기준:

```text
전략이 손실 매도를 자동으로 내지 않아야 한다.
BUY_REJECT, SELL_REJECT 사유가 백테스트와 알림에 표시되어야 한다.
```

## 2단계: 백테스트 진단 강화

- [ ] 사이클별 손익을 출력한다.
- [ ] 사이클별 시작일, 종료일, 누적 매수금, 최종 손익을 출력한다.
- [ ] 미청산 포지션의 평가손익을 별도로 표시한다.
- [ ] 매수거부/매도거부 발생 횟수를 출력한다.
- [ ] 상태 전환 이력을 CSV 또는 JSONL로 저장한다.
- [ ] 수수료를 반영할 수 있는 옵션을 추가한다.
- [ ] 세금 또는 거래비용 옵션을 추가한다.
- [ ] Buy & Hold 대비 초과수익을 종목별/기간별 표로 비교한다.
- [ ] TQQQ, SOXL 2025 H1/H2, 2026 H1 비교 실행 스크립트를 추가한다.

완료 기준:

```text
수익률이 낮거나 거래가 없을 때 원인을 출력만 보고 판단할 수 있어야 한다.
```

## 3단계: 무한매수 V4 핵심 기능 구현

- [ ] 큰수매수 조건을 구현한다.
- [ ] 큰수매수 1회 최대 한도를 구현한다.
- [ ] 큰수매수 누적 최대 한도를 구현한다.
- [ ] 큰수매수 금지 조건을 구현한다.
- [ ] 스킵 상태 진입 조건을 구현한다.
- [ ] 스킵 상태에서 신규 매수 제한을 구현한다.
- [ ] 리버스모드 진입 조건을 구현한다.
- [ ] 리버스모드 매도 기준을 구현한다.
- [ ] 리버스모드 종료 조건을 구현한다.
- [ ] EXIT_WAIT 상태를 구현한다.
- [ ] 상태별 허용 주문 유형을 표로 정리하고 코드에 반영한다.

완료 기준:

```text
문서의 V4 상태 머신이 코드와 일치해야 한다.
READY, NORMAL_FRONT, NORMAL_BACK, BUY_REJECT, SELL_REJECT, BIG_BUY, SKIP, REVERSE, EXIT_WAIT, CLOSED가 모두 의미 있게 동작해야 한다.
```

## 5단계: 실거래 운영 안정성

- [ ] 체결 여부를 확인하고 상태 저장소에 반영한다.
- [ ] 부분체결을 처리한다.
- [ ] 미체결 주문 만료 또는 취소 상태를 처리한다.
- [ ] 동일 clientOrderId 중복 제출 방지를 강화한다.
- [ ] 주문 실패 시 재시도 가능 오류와 중단 오류를 구분한다.
- [ ] API rate limit 발생 시 백오프 정책을 적용한다.
- [ ] 토큰 발급 실패, IP 제한 오류, 권한 오류 알림을 분리한다.
- [ ] 장 마감 후 브로커 상태와 내부 상태를 자동 대조한다.
- [ ] 상태 불일치 시 MANUAL_HALT로 전환한다.

완료 기준:

