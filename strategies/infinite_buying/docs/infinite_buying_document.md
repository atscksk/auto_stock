# 무한매수 V4 기반 자동매매 전략 문서 v0.2

> 기반 자료: 사용자가 업로드한 무한매수 V4 일반모드, 스킵 후 리버스모드, 큰수매수, 매수거부·매도거부 자료, 토스증권 OpenAPI 문서  
> 대상 구현: 토스증권 OpenAPI 기반 JavaScript 자동매매 프로그램  
> 대상 종목: TQQQ 우선, SOXL은 2차 확장  
> 주의: 본 문서는 투자 수익을 보장하지 않는다. 실거래 전 백테스트, 페이퍼 트레이딩, 소액 검증이 필요하다.

---

## 1. 전략 개요

본 전략은 사용자가 제공한 **무한매수법 V4.0** 자료를 기반으로 하되, 자동매매 환경에서 발생할 수 있는 주문 중복, 일부 체결, API 지연, 원금 소진, 장기 하락장 위험을 줄이기 위해 보수적 안정장치를 추가한 실행 전략이다.

기본 흐름은 다음과 같다.

```text
새 사이클 시작
→ 일반모드 LOC 분할매수
→ ★% 기준 LOC 매도
→ 15% 보조 지정가 매도
→ 매수거부/매도거부 판단
→ 불리한 구간에서는 스킵
→ 필요 시 리버스모드
→ 사이클 종료 또는 재시작
```

자동매매 구현에서는 원본 전략의 핵심 구조를 유지하되, 다음 원칙을 우선한다.

```text
1. 수익성보다 상태 동기화와 주문 안정성을 우선한다.
2. 내부 계산값보다 브로커 실제 체결·잔고 데이터를 우선한다.
3. 일부 체결과 미체결을 정상 상황으로 간주하고 처리한다.
4. 마감 직전 주문 취소·정정은 제한한다.
5. 불확실하거나 데이터 불일치가 있으면 자동매매를 중단한다.
```

## 2. 운용 대상

### 2.1 기본 종목

```text
1차 대상: TQQQ
2차 대상: SOXL
```

초기 자동매매는 **TQQQ 1종목만** 사용한다. SOXL은 변동성이 더 크기 때문에 TQQQ에서 최소 1개월 이상 오류 없이 운용된 뒤 추가한다.

### 2.2 운용 순서

```text
1. TQQQ 주문 계산기 모드
2. TQQQ 페이퍼 트레이딩
3. TQQQ 반자동 주문
4. TQQQ 소액 자동매매
5. SOXL 계산기 모드
6. SOXL 소액 확장
```

## 3. 핵심 자금관리

### 3.1 기본 분할

원본 전략의 40분할 구조를 기본으로 사용한다.

```text
전략 총자금 = 사용자가 지정한 USD 기준 운용금
1유닛 = 전략 총자금 / 40
```

예시:

```text
전략 총자금 = 10,000 USD
1유닛 = 10,000 / 40 = 250 USD
```

### 3.2 회차 q와 진행 회차 T

전략 계산에는 두 개의 회차 개념을 사용한다.

```text
q = 전체 회차 기준값
T = 현재 진행 회차
```

기본값:

```text
q = 20
분할 수 = 40
```

중요한 점은 **T를 날짜 수나 주문 횟수로 보지 않고, 누적 투입금액 기준 진행률로 정의한다**는 것이다.

```text
T = 사이클 누적 매수 체결금액 / 1유닛
```

따라서 T는 정수가 아니라 소수점이 될 수 있다.

예시:

```text
전략 총자금 = 10,000
1유닛 = 250
사이클 누적 매수 체결금액 = 2,150

T = 2,150 / 250 = 8.6
```

이 방식은 일부 체결, 1주 단위 체결, 사다리 주문 일부 체결을 자연스럽게 반영한다.

## 4. 브로커 상태 동기화 원칙

자동매매에서 가장 중요한 원칙은 **내부 상태보다 토스증권 계좌의 실제 상태를 우선한다**는 것이다.

### 4.1 매일 장마감 후 동기화

장마감 후 반드시 다음 순서로 상태를 재계산한다.

```text
1. 주문 내역 조회
2. 체결 내역 조회
3. 실제 보유수량 조회
4. 실제 평균단가 조회
5. 매도 가능 수량 조회
6. 사이클 누적 매수 체결금액 재계산
7. T = 누적 매수 체결금액 / 1유닛 재계산
8. 내부 상태와 브로커 상태 비교
9. 불일치 시 MANUAL_HALT
```

### 4.2 상태 불일치 처리

다음 상황에서는 자동 주문을 중단한다.

```text
브로커 보유수량과 내부 보유수량 불일치
브로커 평균단가와 내부 평균단가 차이가 허용범위 초과
체결 내역 누락
미체결 주문 상태 확인 실패
주문 취소 여부 확인 실패
T 계산 불가능
```

상태 불일치 발생 시:

```text
state = MANUAL_HALT
신규 주문 금지
기존 주문 조회만 허용
사용자 수동 확인 필요
```

## 5. 핵심 계산식

### 5.1 ★% 계산

사용자가 제공한 예시의 구조를 따른다.

```text
★% = q - 2 × T
```

예시:

```text
q = 20
T = 8.6

★% = 20 - 2 × 8.6 = 2.8%
```

### 5.2 ★% 가격

```text
★% 가격 = 현재 평단 × (1 + ★% / 100)
```

예시:

```text
현재 평단 = 38.30
★% = 2.8

★% 가격 = 38.30 × 1.028 = 39.37
```

### 5.3 15% 보조 매도가

```text
15% 보조 매도가 = 장마감 후 확정 평단 × 1.15
```

15% 보조 매도가는 장중에 평단이 바뀌더라도 즉시 재계산하지 않는다. 다음 거래일 주문 생성 시점에 재계산한다.

## 6. 전략 상태

전략은 상태머신으로 운용한다.

```text
READY
NORMAL_FRONT
NORMAL_BACK
BUY_REJECT
SELL_REJECT
BIG_BUY
SKIP
REVERSE
EXIT_WAIT
CLOSED
MANUAL_HALT
```

| 상태 | 의미 |
|---|---|
| READY | 새 사이클 시작 전 |
| NORMAL_FRONT | 일반모드 전반전 |
| NORMAL_BACK | 일반모드 후반전 |
| BUY_REJECT | 매수거부 상태 |
| SELL_REJECT | 매도거부 상태 |
| BIG_BUY | 큰수매수 조건 발생 |
| SKIP | 일반 매수 보류 |
| REVERSE | 리버스모드 |
| EXIT_WAIT | 매도 대기 |
| CLOSED | 사이클 종료 |
| MANUAL_HALT | 자동매매 중단, 사용자 확인 필요 |

## 7. 일반모드

### 7.1 새 사이클 시작 조건

```text
보유수량 = 0
미체결 주문 없음
장 운영일
MANUAL_HALT 아님
추세 필터 통과
최소 현금 보존 조건 충족
```

### 7.2 전반전

조건:

```text
T <= q / 2
state = NORMAL_FRONT
```

전반전은 아직 투입금이 적고 자금 여력이 있으므로 일반모드 매수를 정상적으로 수행한다.

주문 구성:

```text
1. ★or큰수 LOC 매수 후보
2. 평단 기준 LOC 매수 후보
3. 하단 사다리 LOC 매수 후보
4. ★% LOC 매도
5. 15% DAY 보조 매도
```

자동매매 제한:

```text
전반전 1일 최대 매수금액 <= 2유닛
큰수매수 포함 시 <= 3유닛
현금 보존선 하회 금지
```

### 7.3 후반전

조건:

```text
T > q / 2
state = NORMAL_BACK
```

후반전은 원금 소진 위험이 커지는 구간이므로 매수 강도를 낮춘다.

주문 구성:

```text
1. 큰수 또는 ★% 기준 LOC 매수 후보
2. 제한적 하단 사다리 LOC 매수
3. ★% LOC 매도
4. 15% DAY 보조 매도
```

자동매매 제한:

```text
후반전 1일 최대 매수금액 <= 1유닛
큰수매수 포함 시 <= 1.5유닛
현금 20% 미만이면 신규 매수 금지
```

## 8. 매도 전략

### 8.1 ★% LOC 매도

기본 매도는 ★% 가격을 기준으로 한다.

```text
매도가 = 평단 × (1 + ★% / 100)
주문 방식 = LOC 매도
주문 매핑 = LIMIT + CLS
```

### 8.2 15% 보조 매도

15% 보조 매도는 고수익 탈출용 보조 주문이다.

```text
매도가 = 장마감 후 확정 평단 × 1.15
주문 방식 = DAY 지정가 매도
```

토스 OpenAPI에서 해외주식 GTC 지원이 명확히 확인되기 전까지는 **15% 보조 매도를 DAY 주문으로 간주한다.**

### 8.3 15% DAY 주문 관리 정책

15% 보조 매도는 다음 정책을 따른다.

```text
1. 매일 주문 생성 시점에 새로 계산한다.
2. 기준 평단은 직전 장마감 후 확정된 평균단가다.
3. 장중 추가 매수 체결로 평단이 변경되어도 당일 즉시 재계산하지 않는다.
4. 다음 거래일 주문 생성 전 기존 15% 주문이 남아 있으면 취소 후 재주문한다.
5. 보유수량이 변경되면 다음 주문 생성 시 매도 가능 수량 기준으로 재산정한다.
6. 15% 보조 매도는 ★% LOC 매도보다 낮은 우선순위를 가진다.
```

## 9. 큰수매수

큰수매수는 일반 1주 사다리보다 더 큰 수량을 특정 조건에서 배치하는 예외 매수다.

자동매매에서는 기본적으로 비활성화하고, 페이퍼 트레이딩 및 소액 검증 후 활성화한다.

### 9.1 발동 조건

```text
현재가가 평단보다 충분히 낮음
★% 가격 또는 큰수 기준 가격이 현재가보다 유리함
남은 현금이 충분함
매수거부 상태가 아님
시장 위험 필터가 정상
```

### 9.2 제한 조건

```text
큰수매수 1회 최대 = 남은 현금의 20%
큰수매수 누적 최대 = 전략자금의 30%
후반전 큰수매수 수량 = 전반전 대비 50% 축소
같은 사이클 내 큰수매수 최대 2회
```

### 9.3 금지 조건

```text
최근 5거래일 중 4거래일 하락
현재가 < 200일 이동평균
당일 전일 종가 대비 -8% 이하
잔여 현금 < 25%
평가손실 -30% 이하
```

## 10. 매수거부

매수거부는 계산상 매수 가격이 나오더라도 자동매수하지 않는 방어 규칙이다.

### 10.1 조건

```text
매수 LOC 가격이 현재가 대비 지나치게 높음
매수 후 평균단가 개선 효과가 작음
보유수량이 이미 충분함
매수 가능 현금 부족
하락 추세가 강함
현금 보존선 침범
```

### 10.2 실행

```text
신규 매수 주문 생성 안 함
큰수매수 금지
기존 매도 주문은 유지 또는 재계산
스킵 또는 리버스모드 진입 조건 확인
```

---

## 11. 매도거부

매도거부는 계산상 매도 가격이 나오더라도 전략상 불리하다고 판단되면 매도를 보류하는 규칙이다.

### 11.1 조건

```text
매도 수익률이 지나치게 낮음
매도 후 재진입 가격이 불리함
보유수량 대비 매도 수량이 비효율적
큰수매수 직후라 평균단가 개선 효과를 기다려야 함
리버스모드 매도 기준이 우선 적용됨
```

### 11.2 실행

```text
★% LOC 매도 보류
15% DAY 보조 매도는 유지 가능
다음 거래일 재계산
```

단, 매도거부가 과도하게 길어지는 것을 막기 위해 다음 제한을 둔다.

```text
매도거부 연속 5거래일 초과:
- 최소 30% 보유분은 ★% LOC 매도 시도
- 또는 MANUAL_HALT 전환
```

## 12. 스킵 후 리버스모드

### 12.1 스킵 조건

```text
일반 매수 조건은 있으나 매수거부 발생
현금 부족
가격 급락
현재가가 주요 이동평균 아래
큰수매수 과다 실행
브로커 상태 불일치 전조
```

### 12.2 리버스모드 진입

```text
state = SKIP
반등 기준 또는 리버스 기준 가격 계산 가능
state = REVERSE
```

### 12.3 목적

```text
추가 물타기 제한
반등 시 보유분 정리
손실 축소 또는 작은 수익으로 탈출
새 사이클 준비
```

### 12.4 종료 조건

```text
보유수량 = 0
보유수량이 사이클 최대 보유수량 대비 30% 이하
평단 대비 수익 전환
사용자 강제 종료
```

종료 후:

```text
CLOSED
→ 1거래일 휴식
→ READY
```

## 13. 주문 타이밍 정책

LOC 전략은 장마감 부근 주문이 중요하지만, 마감 직전 취소·정정은 API 지연이나 브로커 제한 위험이 있다. 따라서 보수적 cutoff 정책을 둔다.

### 13.1 기본 시간 정책

```text
미국장 마감 60분 전: 1차 주문계획 생성
미국장 마감 45분 전: 기존 미체결 주문 정리
미국장 마감 35분 전: 신규 LOC 주문 제출
미국장 마감 20분 전: 주문 접수 여부 확인
미국장 마감 15분 전 이후: NO_TOUCH
미국장 마감 후: 체결 확인만 수행
```

### 13.2 설정값

```text
CANCEL_CUTOFF_MINUTES_BEFORE_CLOSE = 45
ORDER_CUTOFF_MINUTES_BEFORE_CLOSE = 35
NO_TOUCH_MINUTES_BEFORE_CLOSE = 15
```

### 13.3 NO_TOUCH 구간

NO_TOUCH 구간에서는 다음 작업을 금지한다.

```text
신규 주문
주문 취소
주문 정정
중복 재시도
주문계획 변경
```

허용 작업:

```text
주문 상태 조회
위험 로그 기록
사용자 알림
```

## 14. 위험관리 필터

### 14.1 현금보존

```text
최소 현금 비율 = 20%
초기 권장 현금 비율 = 30%
현금보존선 하회 시 신규 매수 금지
```

### 14.2 종목별 최대 투입

```text
TQQQ 최대 투입 = 전략자금의 50%
SOXL 최대 투입 = 전략자금의 30%
```

### 14.3 추세 필터

```text
TQQQ:
- QQQ가 200일 이동평균 아래면 신규 사이클 금지
- 기존 사이클은 매수 규모 50% 축소

SOXL:
- SOXX 또는 SMH가 200일 이동평균 아래면 신규 사이클 금지
- 기존 사이클은 큰수매수 금지
```

외부 ETF 데이터가 확보되지 않으면 해당 종목 자체의 200일 이동평균을 대체로 사용한다.

### 14.4 급락 필터

```text
당일 현재가가 전일 종가 대비 -8% 이하:
- 신규 매수 금지
- 하단 LOC 매수 취소
- 매도 주문만 유지
```

### 14.5 연속 하락 필터

```text
최근 5거래일 중 4거래일 하락:
- 큰수매수 금지
- 일반매수 수량 50% 축소
```

### 14.6 평가손실 중단

```text
평가손실 -30%:
- 신규 매수 중단
- 리버스모드 검토

평가손실 -45%:
- MANUAL_HALT
- 사용자 승인 전 주문 금지

평가손실 -60%:
- 자동매매 완전 중지
- 신규 주문 금지
```

## 15. 주문 우선순위

하루 주문 생성 우선순위는 다음으로 고정한다.

```text
1. MANUAL_HALT 여부
2. 장 운영 여부
3. NO_TOUCH 구간 여부
4. 브로커 상태 동기화 결과
5. 미체결 주문 확인
6. 보유수량·현금 확인
7. 매도거부 판단
8. 매수거부 판단
9. 리버스모드 판단
10. 스킵 판단
11. 큰수매수 판단
12. 일반모드 매수 판단
13. ★% LOC 매도 생성
14. 15% DAY 보조 매도 생성
15. 주문 가능 금액·수량 최종 검증
```

핵심 원칙:

```text
MANUAL_HALT > 모든 주문
NO_TOUCH > 모든 주문 변경
브로커 실제 상태 > 내부 상태
현금보존 규칙 > 모든 신규매수
매수거부 > 큰수매수
리버스모드 > 일반모드
★% LOC 매도 > 15% DAY 보조 매도
```


### 17.3 장마감 전

```text
1. 주문 cutoff 시간 확인

3. 필요한 경우 cutoff 이전에 취소
4. 최종 주문계획 생성
5. 매수거부/매도거부 확인

7. 15% DAY 보조 매도 제출
```

### 17.4 NO_TOUCH 구간

```text

```

### 17.5 장마감 후

```text

2. 실제 보유수량 조회

6. 내부 상태와 브로커 상태 대조
```

## 20. 최종 결론

본 전략은 원본 무한매수 V4의 구조를 유지하되, 자동매매에 맞게 다음 요소를 강화한 형태다.

```text
T 소수점 회차 처리
브로커 상태 재동기화
일부 체결 대응
주문 cutoff 정책
NO_TOUCH 구간
15% DAY 주문 재생성 정책
MANUAL_HALT 안전장치
현금보존과 추세필터
큰수매수 제한
리버스모드 방어 운용
```

최종 목표는 공격적인 자동매매가 아니라, **계산 가능한 주문만 실행하고, 불확실하면 멈추는 보수형 자동매매 시스템**이다.
